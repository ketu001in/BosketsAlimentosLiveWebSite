================================================================================
GLOBAL RULES
================================================================================
- 100% vegetarian content only — no meat, no eggs in any recipe context.
- All pages must be responsive (mobile-first), support dark mode, and be
  SEO-optimised with appropriate meta tags.
- CSRF protection on every POST form. Sessions use SameSite=Lax cookies.
- All file uploads validated server-side (type + size). Images max 5 MB,
  videos max 25 MB.
- No frameworks on the backend — pure PHP 8.3 + PDO.
- Schema changes use ensure_*() idempotent migration functions — never
  require manual SQL or re-running install.php.
- Do NOT commit: .env, keystore files, adminer.php, diagnostic scripts,
  or any credentials.
- Before building any module, ask clarifying questions until 99% sure.

================================================================================
MODULE 1 — USER MANAGEMENT
================================================================================

1.1 Registration & Account
    - Register with username, display name, email, password (bcrypt hashed).
    - Optional profile picture upload (avatar) — max 5 MB, JPG/PNG/WEBP/GIF.
    - Email uniqueness enforced; username uniqueness enforced (alphanumeric +
      underscore, 3–30 chars).
    - Optional newsletter opt-in checkbox at registration (email_notify flag +
      unique unsubscribe token generated per user).
    - Full account management: login, logout, password reset via email token,
      change password, edit profile (display name, bio, email, avatar),
      delete account (cascades all user content).
    - Social login: Google OAuth + Facebook OAuth (optional — only shown when
      OAuth keys configured in config.php). Users can link/unlink providers.

1.2 Profile & Wall
    - Each user has a profile page with tabs: Wall | Recipes | Buddies.
    - Wall: user can post text updates and share recipes.
    - All site visitors can view profiles; only logged-in users can react/comment.
    - Profile shows avatar, display name, username, bio, join date, buddy count.

1.3 Reactions & Comments
    - Reactions on recipes, wall posts and forum topics: like 👍, love ❤️,
      yum 😋, wow 🤩 (one per user per item, toggleable).
    - Comments on recipes, wall posts and forum topics.
    - Comments moderated post-publish (admin can hide/restore/delete).

1.4 Buddy System
    - Send buddy request → recipient gets in-app notification → accept/reject.
    - Accepted buddies appear in each other's Buddy List.
    - Buddy Feed: shows all activity from accepted buddies.
    - Find Buddies page to discover other members.

1.5 Private Messaging
    - One-on-one real-time-style chat between buddy-connected users.
    - Unread message count in header nav.
    - Messages page shows conversation threads.

1.6 Notifications (In-App)
    - Bell icon in header shows unread count (notifications + announcements).
    - Notification types: buddy request, recipe approved/rejected, comment,
      reaction, message received, system announcement.
    - Notifications page: shows all notifications + unread admin announcements.
    - Per-item Mark as Read + Dismiss. Mark All as Read.
    - Announcements from admin persist until user dismisses them.

================================================================================
MODULE 2 — RECIPE MANAGEMENT
================================================================================

2.1 Recipe Submission (Post a New Recipe)
    Non-admin submissions go to status = pending (hidden from public until
    admin approves). Admin submissions go live immediately.

    SECTION 1 — Basic Details
      - Recipe Name (mandatory, max 150 chars).
      - Main picture (mandatory on create; optional on edit). JPG/PNG/WEBP/GIF,
        max 5 MB.
      - Short description / story behind the dish (optional, max 3000 chars).
      - YouTube video link (optional — embeds on recipe page + adds VideoObject
        to JSON-LD schema).

    SECTION 2 — Ingredients (mandatory — minimum 1)
      - Ingredient name + quantity per row. Add/remove rows dynamically.
      - Typeahead autocomplete (3-letter trigger) against master ingredients list.
      - New ingredient names auto-added to master list for future use.

    SECTION 3 — General Information (optional)
      - Category, Cuisine Type, Origin — all typeahead with master lists.
      - Prep Time (minutes) — optional, used in SEO schema (ISO 8601 format).
      - Cook Time (minutes) — optional, used in SEO schema.

    SECTION 4 — Step-by-Step Method (mandatory — minimum 1 step)
      - Each step: instruction text + optional photo or short video (max 25 MB).
      - Steps can be added, reordered, removed.

    SECTION 5 — Verdict / Secrets / Trivia (optional, max 3000 chars)

    On submit (non-admin): flash "Your recipe is pending admin approval."
    On submit (admin): goes live immediately + email notifications sent.

2.2 Recipe Display Page
    - Hero: 4:3 image frame (object-fit: cover). Click image → fullscreen
      lightbox with fade-in animation (Esc or click outside to close).
    - Blurred background fill behind the contained image using second <img>.
    - YouTube video embedded below hero if URL provided.
    - Ingredients list, step-by-step collapsible method, verdict section.
    - Reactions bar + comments section.
    - Share modal: WhatsApp, Facebook, X/Twitter, copy link, repost to wall.
    - Edit/Delete buttons for recipe owner and admins.

2.3 Recipe Listing & Search
    - Filterable by category, cuisine, origin.
    - Live search by recipe name (AJAX, header search bar).
    - Pagination (12 per page default).
    - Excludes pending and removed recipes for public visitors.

2.4 Recipe Card
    - 16:10 image, recipe title, category · cuisine meta, author avatar + name,
      reaction count, comment count.
    - Featured recipes marked with orange star badge.

MODULE 5 — ADMIN PANEL (/admin/)
================================================================================
Admin panel is accessible only to users with is_admin = 1.

5.1 Dashboard (/admin/index.php)
    - Stats: Members, Published Recipes, Forum Topics, Comments, Reactions,
      Buddy Pairs, Wall Posts, Ingredients.
    - Newest members list, latest recipes list.

5.2 Content Moderation (/admin/content.php) — Tabbed interface
    Tab 1 — PENDING APPROVAL
      - All pending user-submitted recipes with thumbnail, title, author,
        category, cuisine, submit time, story excerpt.
      - Preview button (opens recipe in new tab).
      - Approve button → recipe goes live + in-app notification + email
        notifications sent to subscribers.
      - Reject button → shows textarea for optional reason → recipe removed +
        in-app notification with reason sent to author.

    Tab 2 — RECIPES
      - All published/hidden recipes with title, author, views, status.
      - Actions per recipe: Feature / Un-feature, ⭐ Set as Star Recipe,
        Hide / Restore, Permanent Delete.

    Tab 3 — COMMENTS
      - All comments (recipe + forum + wall) with body, author, target.
      - Actions: Hide / Restore / Delete.

    Tab 4 — FORUM TOPICS
      - All topics with title, author, views, replies, status.
      - Actions: Hide / Restore / Delete (cascades replies).

    Tab 5 — WALL POSTS
      - All wall posts with body, author, comment count, status.
      - Actions: Hide / Restore / Delete (cascades comments).

    Star Recipe Settings panel (above tabs):
      - Label text field (default "Star Recipe" — admin changes to
        "Recipe of the Week", "Dish of the Day", etc.)
      - Mode: Auto (highest score this week) or Manual (admin picks).
      - Auto score = (reactions in last 7 days × 3) + (total views / 10).

5.3 Users (/admin/users.php)
    - List all users with username, email, join date, is_admin, is_banned.
    - Ban/Unban, Promote to Admin / Demote, View Profile.

5.4 Announcements (/admin/announcements.php)
    - Create push announcement: Title (required) + Message Body.
    - Sent to ALL registered users — appears in their notification bell.
    - History list: shows all announcements with read count, dismissed count.
    - Hard Delete from DB (removes announcement + all read records).
    - DB tables: announcements (id, title, body, created_by, created_at,
      is_deleted) + announcement_reads (user_id, announcement_id,
      is_dismissed, created_at).

5.5 Gallery (/admin/gallery.php)
    - Upload up to 200 images (multi-select supported).
    - Set/edit caption per image. Delete individual images.
    - Images display on About Us page in masonry grid.
    - DB table: gallery (id, image, caption, sort_order, created_at, created_by).

5.6 Import Tool (/admin/import-blogspot.php)
    - Bulk import from bosketsalimentos.blogspot.com Blogger JSON feed.
    - Paginated fetch (50 per batch). Duplicate detection by title.
    - Downloads images from Blogger CDN to uploads/recipes/.
    - Maps Blogspot labels → category/cuisine/origin taxonomy.
    - Saves as draft. Admin reviews and publishes.

5.7 Image Repair (/admin/repair-images.php)
    - Re-downloads missing recipe images from Blogspot feed.
    - Matches posts to DB recipes by title. Updates recipes.image DB field.

5.8 CMS — Pages (/admin/cms-pages.php)
    - Create, edit, publish, delete static pages with HTML textarea editor.
    - Fields: title, slug (auto-generated), body HTML, meta description,
      visibility (public/members), status (draft/published).
    - Auto-seeds "About Us" and "Contact Us — Intro" pages on first visit.
    - Published pages render at /page.php?slug=... with full site layout.

5.9 CMS — Appearance (/admin/cms-appearance.php)
    - Default theme: Light / Dark.
    - Visitor toggle: show/hide light-dark toggle button for site visitors.
    - Font pairing: Playfair Display + Inter (default) or 4 other options.
    - Brand colours: primary green + accent orange (output as CSS :root
      variable overrides in <head> — applies site-wide instantly).
    - Settings stored in cms_settings table (key-value).

5.10 CMS — Menus (/admin/cms-menus.php)
    - Add/delete/reorder navigation and footer menu items.
    - Link type: CMS page or custom URL.
    - Open in new tab option.
    - Locations: top navigation bar + footer.

5.11 Master Lists (/admin/masters.php)
    - Manage ingredients, categories, cuisines, origins master lists.
    - Add, rename, merge.

5.12 Messages (/admin/messages.php)
    - View contact form submissions.

MODULE 6 — STAR RECIPE (Homepage Feature)
================================================================================
- star_recipe DB table (one row): recipe_id, label, mode, updated_at, updated_by.
- Auto mode: every page load computes highest-scoring recipe.
  Score = (reactions in last 7 days × 3) + (total views / 10).
  Orders DESC, picks top 1 published recipe.
- Manual mode: admin clicks "⭐ Set Star" on any recipe in admin/content.php.
- Homepage ribbon: slim dark strip (~80px) with blurred thumbnail,
  ⭐ badge + admin-configurable label, recipe title, category/cuisine tags,
  author name, "View Recipe →" button. Responsive (collapses on mobile).

MODULE 7 — EMAIL NOTIFICATIONS
================================================================================
- Opt-in at registration (checkbox) and manageable in Account Settings.
- email_notify TINYINT(1) DEFAULT 0 and email_token VARCHAR(64) on users table.
- Trigger 1: Admin posts recipe → email ALL subscribed users.
- Trigger 2: Any user posts recipe → email subscribed accepted buddies.
- HTML email template: site header, recipe image, title, story excerpt,
  "View Recipe →" button, unsubscribe link.
- From: noreply@bosketsalimentos.com (Hostinger domain email).
- Unsubscribe: /unsubscribe.php?token=XXXX → sets email_notify = 0.
- Users can re-subscribe via Account Settings.

MODULE 8 — SEO
================================================================================

8.1 Recipe JSON-LD (Schema.org/Recipe) — on every recipe page
    Fields: name, description, author (Person), datePublished, keywords,
    recipeIngredient (array), recipeInstructions (HowToStep array),
    recipeCategory, recipeCuisine, image (array of URLs).
    Conditional: prepTime, cookTime, totalTime (ISO 8601 if set),
    VideoObject (if YouTube URL present), aggregateRating (if reactions > 0).

8.2 WebSite Schema — on every page (via cms_head_html())
    WebSite + SearchAction (target: /recipes.php?q={search_term_string}).
    Enables Google sitelinks search box.

8.3 OG / Twitter meta tags — on every page
    og:type = article on recipe pages; website elsewhere.
    og:image, og:title, og:description, og:url on all pages.

8.4 Sitemap (/sitemap.php)
    Dynamic XML: homepage, recipes.php, forum.php, all recipe pages,
    all forum topic pages, about.php, contact.php, privacy-policy.php.
    changefreq and priority per URL type.

8.5 robots.txt
    Disallows: /admin/, /api/, /includes/, settings, notifications, feed,
    buddies, messages, auth pages, install.php, unsubscribe.php.
    Sitemap: https://www.bosketsalimentos.com/sitemap.php (absolute URL).

MODULE 9 — PERFORMANCE (.htaccess)
================================================================================
- mod_deflate: gzip compress HTML, CSS, JS, JSON, SVG, fonts (~70% smaller).
- mod_expires + Cache-Control:
    Images / video / fonts: 6 months (immutable).
    CSS / JS: 1 year (safe — versioned with ?v=N query string).
    PHP pages: no-store, no-cache.
- Security headers: X-Content-Type-Options, X-Frame-Options (SAMEORIGIN),
  Referrer-Policy, Permissions-Policy, X-XSS-Protection.
- Keep-alive header. ETags disabled.
- ErrorDocument 404 /index.php.
- PHP limits: upload_max_filesize 30M, post_max_size 64M, max_execution_time 120.

DEPLOYMENT
================================================================================
Method  : ZIP → hPanel File Manager → Extract (Overwrite existing files)

          MAIL_FROM, Google/Facebook OAuth keys)

          Bump N when deploying CSS changes.

          fresh install (excluded from ZIP).
